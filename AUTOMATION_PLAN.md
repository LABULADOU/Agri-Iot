# Agri-Iot 自动化管理方案

## 目标
让 Hermes Agent 接管 Agri-Iot 项目的日常管理工作，包括：
- 代码维护与 bug 修复
- 知识库更新与管理
- 设备监控与异常处理
- 生产环境日常管理（灌溉、通风、紧急情况）

## 阶段一：代码维护自动化（立即可做）

### 1.1 定时代码审查
- 每天检查代码变更
- 自动运行测试套件
- 发现编译错误或测试失败时通知

### 1.2 依赖更新
- 定期检查 Rust/Cargo 依赖更新
- 检查 npm 包更新
- 提供安全漏洞报告

### 1.3 文档同步
- 代码变更后自动更新 API 文档
- 维护 CHANGELOG

## 阶段二：知识库管理（已完成部分）

### 2.1 知识库维护
- 定期整理 Obsidian 笔记
- 检查知识条目完整性
- 自动更新索引

### 2.2 数据驱动的知识更新
- 从传感器数据中提取规律
- 更新作物生长模型
- 积累调控案例

## 阶段三：设备监控（需部署后）

### 3.1 定时健康检查
- 检查 ESP32 设备在线状态
- 分析传感器数据异常
- 检查数据库状态

### 3.2 异常告警
- 温度/湿度超出阈值
- 设备离线通知
- 数据中断检测

## 阶段四：温室环境管理（需部署后）

### 4.1 灌溉控制
- 根据土壤湿度自动建议灌溉
- 分析灌溉效果
- 优化灌溉计划

### 4.2 通风控制
- 根据温湿度自动建议通风
- 分析通风效果
- 优化通风策略

### 4.3 紧急情况处理
- 高温/低温保护
- 暴雨/大风应对
- 设备故障应急

## 实施计划

### 第一步：配置 Hermes 定时任务
- 设置代码审查 cron job
- 设置知识库检查 cron job
- 设置依赖更新检查 cron job

### 第二步：完善监控脚本
- 编写设备健康检查脚本
- 编写数据异常检测脚本
- 编写温室环境分析报告

### 第三步：部署到生产环境
- 配置服务器环境
- 设置自动化脚本
- 配置告警通知

## 所需配置

### 环境变量
```bash
# LLM API 配置（用于 AI 对话和决策）
LLM_API_KEY=your_api_key
LLM_API_URL=https://api.openrouter.ai/v1
LLM_MODEL=openrouter/anthropic-claude-sonnet-4

# 知识库路径
OBSIDIAN_VAULT_PATH=/home/xuhan/agri-knowledge

# 项目路径
AGRI_IOT_PATH=/home/xuhan/Agri-Iot

# 告警配置
ALERT_WECHAT_CHAT_ID=o9cq8029nrzsAGfY0-17InwEagWw@im.wechat
```

### Cron Jobs
1. **代码审查**: 每天 9:00
2. **知识库检查**: 每天 10:00
3. **依赖更新**: 每周日 2:00
4. **健康检查**: 每 30 分钟（部署后）
5. **环境报告**: 每天 8:00 和 20:00

## 优势

1. **7x24 小时监控**: 无人值守运行
2. **自动响应**: 异常情况及时处理
3. **知识积累**: 越用越聪明
4. **多平台通知**: 微信/Telegram 随时接收报告
